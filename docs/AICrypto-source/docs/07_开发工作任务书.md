# AICrypto 开发工作任务书

## 1. 目标

本任务书用于指导首轮研发执行，确保团队按统一架构推进，并优先完成“币安永续合约自动研究 -> 策略 Skill 化 -> 风控 -> 自动执行”的最小闭环。

## 2. 工作包总览

### WP1：工程与基础设施

负责人：后端架构 / DevOps

任务：

- 建立 monorepo 目录结构
- 初始化 Rust workspace
- 初始化 Python research workspace
- 初始化 Next.js 控制台
- 配置 PostgreSQL、Redis、NATS JetStream
- 配置基础 CI/CD
- 配置日志、Trace、Metrics

交付标准：

- 新成员可按文档启动本地环境
- 各服务具备健康检查

### WP2：币安接入层

负责人：Rust 后端

任务：

- 实现 `exchangeInfo` 同步
- 实现 K 线历史采集
- 实现 Funding/OI/Mark Price 采集
- 实现市场流订阅和重连
- 实现用户流订阅和保活
- 实现测试网下单/撤单/查单
- 建立 Binance Adapter 抽象层

交付标准：

- 测试网全链路跑通
- 数据和订单状态可追溯

### WP3：数据模型与存储

负责人：平台后端

任务：

- 设计 symbol 主表
- 设计 K 线时序表
- 设计 Funding/OI/Mark Price 表
- 设计订单、成交、仓位表
- 设计策略、Skill、回测结果表
- 建立数据保留策略和归档策略

交付标准：

- 支持查询、回放、回测和审计

### WP4：研究与特征工厂

负责人：量化研究 / Python

任务：

- 定义主升浪/主跌浪样本标准
- 开发样本提取脚本
- 开发基础特征库
- 开发相关性分析任务
- 输出首批策略研究报告

交付标准：

- 至少形成 3 类趋势样本研究结论
- 至少形成 2 类相关性规则样本

### WP5：回测引擎

负责人：量化工程 / Rust + Python

任务：

- 定义 Strategy Spec
- 实现订单撮合模拟
- 实现手续费和滑点模型
- 实现回测指标体系
- 实现回测报告导出

交付标准：

- 任意 Skill 可被回测
- 回测结果可复现

### WP6：Skill Registry 与 Signal Engine

负责人：平台后端 / 量化工程

任务：

- 设计 Skill DSL 或配置结构
- 实现 Skill 版本管理
- 实现 Skill 依赖校验
- 实现市场状态引擎
- 实现实时信号评估器

交付标准：

- 至少 10 个 Skill 可加载运行

### WP7：风控引擎

负责人：风控工程 / Rust 后端

任务：

- 实现账户级、标的级、策略级、组合级规则
- 实现熔断、降风险和 reduce-only 通道
- 实现异常订单与异常仓位处理
- 实现风险审计日志

交付标准：

- 下单前必过风控
- 能在异常情况下快速阻断新风险

### WP8：仿真与实盘执行

负责人：交易基础设施

任务：

- 实现仿真账户和虚拟成交逻辑
- 实现实盘订单状态机
- 实现对账任务
- 实现执行质量监控

交付标准：

- 同一策略可在仿真和测试网执行

### WP9：控制台与运营后台

负责人：前端 / 全栈

任务：

- 市场总览页面
- 币种画像页
- Skill 中心
- 回测中心
- 仿真盘中心
- 实盘监控中心
- 风控中心
- 审计中心

交付标准：

- 运营和研究可通过 UI 完成主要操作

### WP10：安全与治理

负责人：平台安全 / 管理后台

任务：

- 密钥管理
- 权限模型
- 审计日志
- 配置变更留痕
- 环境隔离

交付标准：

- 无明文密钥入库
- 敏感操作可追溯

## 3. 首批关键研发清单

### 必须立即启动

- 币安测试网接入
- 合约元数据与行情采集
- 数据库 schema 设计
- 回测引擎骨架
- Skill Registry 骨架
- 风控引擎骨架
- 控制台骨架

### 第一批研究专题

- BTC 突破驱动的市场 risk-on 模式
- BTC 破位驱动的市场 risk-off 模式
- 龙头带动板块补涨模式
- 主升浪二次确认模式
- 主跌浪反抽做空模式
- 极端 meme 事件驱动模式

## 4. 组织协作建议

- 架构负责人每周审查系统边界和接口。
- 研究与工程每周共评 Skill 候选。
- 风控负责人拥有实盘上线否决权。
- 前后端围绕统一领域模型协作，禁止各自定义不同术语。

## 5. 交付纪律

- 所有策略先回测，再仿真，再测试网，再小仓位实盘。
- 所有生产变更必须有回滚方案。
- 所有订单异常必须能复盘到根因。
- 所有策略上线必须绑定版本和报告。

## 6. 建议的下一步动作

1. 初始化仓库目录结构。
2. 先实现币安测试网行情与下单闭环。
3. 同步建立 TimescaleDB schema。
4. 产出主升浪/主跌浪样本定义文档。
5. 实现第一个可回测 Skill：`BTC_breakout_market_regime_long`。
6. 实现第一个风险 Skill：`global_risk_off_reduce_exposure`。
