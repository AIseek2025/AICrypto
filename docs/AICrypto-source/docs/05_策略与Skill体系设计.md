# 策略与 Skill 体系设计

## 1. 核心思想

AICrypto 的竞争力不在于单一指标，而在于把可复用的交易知识拆成标准化 Skill，再由 Agent 在不同市场状态下调用。Skill 不是简单的“开多/开空信号”，而是完整的交易动作单元。

## 2. Skill 的标准定义

一个 Skill 至少包含 7 部分：

1. 适用市场状态
2. 触发条件
3. 禁用条件
4. 入场计划
5. 加减仓计划
6. 止盈止损/退出计划
7. 风险预算与冷却机制

示例类别：

- 趋势突破开仓 Skill
- 回踩确认加仓 Skill
- 抛物线赶顶退出 Skill
- 主跌浪反弹做空 Skill
- 龙头带板块跟随 Skill
- BTC 失守关键位风险收缩 Skill
- 极端消息冲击熔断 Skill

## 3. 主升浪/主跌浪研究框架

## 3.1 主升浪研究对象

主升浪不是局部上涨，而是具备以下特征的连续性趋势阶段：

- 趋势持续时间足够长
- 涨幅远大于日常波动
- 伴随成交放大和关注度提升
- 回撤受控或快速被承接
- 通常有叙事、板块、龙头、事件或 BTC 大盘共振

## 3.2 主跌浪研究对象

主跌浪通常具备：

- 趋势破位
- 恐慌性放量
- 资金撤退和 OI 异动
- 关键支撑连续失守
- 市场状态切换到 risk-off

## 3.3 研究维度

### A. 价格行为

- 突破前高/前低
- 均线结构与斜率
- 高低点抬升/下降
- 回撤深度与恢复速度
- 波动收缩后扩张

### B. 成交与流动性

- 成交额变化
- 换手提升
- 大单成交
- 深度与滑点结构

### C. 衍生品结构

- Funding 持续正/负偏离
- OI 增长是否配合价格
- Basis 扩大或收缩
- 强平簇推断

### D. 叙事与板块

- 所属赛道热度提升
- 龙头币率先启动
- 同板块联动增强
- 事件催化持续发酵

### E. 社交传播

- 名人发声
- meme 扩散
- 推文传播速度
- 社区情绪极化

### F. 市场环境

- BTC 是否突破新高/跌破关键位
- 总风险偏好高低
- 山寨季强弱
- 宏观风险事件冲击

## 4. 相关性策略框架

## 4.1 相关性类型

1. 大盘主导相关
   - BTC -> 全市场风险偏好
2. 板块轮动相关
   - 龙头币 -> 同赛道二三线标的
3. 叙事相关
   - AI、L2、DeFi、Meme 等叙事扩散
4. 人物影响相关
   - Elon Musk、CZ、Vitalik 等
5. 宏观事件相关
   - 战争、政策、流动性冲击、ETF 进展
6. 链上/项目事件相关
   - 解锁、空投、升级、黑客事件、上币消息

## 4.2 相关性知识图谱

建议建立一张可持续更新的图谱：

- 节点：币种、人物、机构、赛道、事件、关键词
- 边：历史联动强度、时滞、方向性、置信度、衰减速度

这样在“某事件出现”时，系统不是死板规则匹配，而是沿知识图谱找到潜在受影响标的与方向。

## 5. 市场状态机设计

建议先定义大盘状态：

- `BULL_TREND`
- `BULL_EUPHORIA`
- `RANGE_NEUTRAL`
- `RISK_OFF`
- `PANIC_SELL`
- `SHORT_SQUEEZE`
- `EVENT_DRIVEN`

每个状态下只允许一部分 Skill 生效，避免在错误环境里调用正确策略。

## 6. Agent 体系设计

### 6.1 Research Agent

职责：

- 扫描历史主升浪/主跌浪样本
- 提炼特征组合
- 发现可编码的规律
- 输出 Skill Proposal

### 6.2 Signal Agent

职责：

- 实时扫描市场
- 判断当前是否满足 Skill 前置条件
- 生成信号和置信度

### 6.3 Execution Agent

职责：

- 把意图翻译为具体下单计划
- 处理挂单、追单、分批建仓、减仓

### 6.4 Risk Agent

职责：

- 实时审核可否交易
- 判断是否需要减仓、暂停或熔断

### 6.5 Review Agent

职责：

- 交易后复盘
- 评估 Skill 实际表现是否偏离预期
- 给出降权、升级或下架建议

## 7. Skill 运行流程

1. 市场状态引擎输出 regime。
2. Signal Agent 对每个 symbol 扫描候选 Skill。
3. 满足前置条件的 Skill 进入候选池。
4. 组合引擎按收益风险比、相关暴露和优先级做筛选。
5. Execution Agent 生成交易计划。
6. Risk Agent 审核。
7. 通过后执行。
8. 订单、仓位、PnL、风险结果回流到 Skill 评价系统。

## 8. 首批建议落地的 Skill 组

### 8.1 趋势组

- BTC 突破历史关键位带动市场 risk-on
- 山寨币平台突破主升浪启动
- 龙头二次回踩确认继续趋势
- 高位加速后尾部离场

### 8.2 做空组

- BTC 关键支撑失守后的全市场 risk-off
- 山寨破位反抽做空
- 负面事件触发的趋势下跌跟随
- 高 funding + 弱价格结构的下杀回归

### 8.3 相关性组

- 龙头启动后的同板块补涨
- meme 传播热度爆发后的相关币交易
- 名人发言后的事件驱动交易

### 8.4 风控组

- 极端波动自动降低杠杆
- 同板块暴露过高自动缩仓
- 消息源置信度不足禁止开仓
- 市场 regime 切换时清理不匹配仓位

## 9. 回测与验证方法

每个 Skill 必须通过以下四关：

1. 历史样本解释性验证
2. 跨周期回测
3. 事件回放/仿真交易
4. 小仓位实盘灰度

必须输出：

- 胜率
- 盈亏比
- 最大回撤
- 收益波动比
- 不同市场状态表现
- 连续亏损分布
- 滑点与手续费影响
- 失效条件

## 10. 为什么要把策略封装成 Skill 而不是散落在代码里

- 便于版本管理
- 便于回测对照
- 便于 Agent 调用
- 便于灰度发布
- 便于审计和追责
- 便于后续建立 Skill 市场与组合引擎
