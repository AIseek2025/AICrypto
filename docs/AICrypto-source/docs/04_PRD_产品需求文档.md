# AICrypto PRD 产品需求文档

## 1. 产品概述

AICrypto 是面向加密货币永续合约市场的 AI 自动化投研及交易平台。产品主要服务于内部研究员、策略开发者、风险管理者和交易运营者，用于从“研究发现机会”走到“自动执行交易并持续复盘优化”的完整闭环。

## 2. 产品目标

- 覆盖币安永续合约全市场研究。
- 构建主升浪/主跌浪发现系统。
- 构建相关性事件驱动交易系统。
- 通过 Skill 与 Agent 机制沉淀可复用交易能力。
- 提供可回测、可仿真、可实盘灰度的平台化产品能力。

## 3. 用户角色

### 3.1 研究员

职责：研究行情规律、总结因子、定义策略、输出 Skill 草案。

### 3.2 策略工程师

职责：把研究结论工程化，实现策略、回测、评估与版本管理。

### 3.3 风控官

职责：配置风控规则、审核策略上线、监控异常。

### 3.4 交易运营

职责：观察实盘运行、处理告警、执行应急操作。

### 3.5 管理员

职责：权限配置、密钥管理、环境开关、审计查看。

## 4. 核心使用场景

1. 查看全市场永续合约扫描结果，发现可能进入趋势启动的币种。
2. 查看某个币种的基础画像、历史特征、资金结构、相关事件与策略建议。
3. 把某个研究结论封装成 Skill 并提交回测。
4. 让 Agent 在纸面盘或实盘中自动调用符合条件的 Skill。
5. 在控制台实时观察策略、仓位、订单、PnL、风险暴露和告警。
6. 出现异常时一键降低风险或全局熔断。

## 5. 产品模块

### 5.1 市场总览

功能：

- 全永续合约列表
- 涨跌幅排行
- 成交量排行
- OI 变化排行
- Funding 异常排行
- 波动率排行
- 板块/赛道热度排行
- 市场状态总评分

验收标准：

- 1 秒到 1 分钟频段的数据可以刷新。
- 支持按交易活跃度、波动性、叙事赛道筛选。

### 5.2 币种画像页

功能：

- 基本信息
- 多周期 K 线
- Mark Price / Funding / OI
- 量价结构
- 相关币与相关事件
- 风险标签
- 当前可调用 Skill 建议
- 历史信号回放

验收标准：

- 任一永续合约都可以打开画像页。
- 图表和特征数据可追溯到原始数据。

### 5.3 市场状态引擎面板

功能：

- BTC 主导状态
- ETH 跟随状态
- 山寨风险偏好评分
- Meme 情绪评分
- 板块轮动热度
- 风险开关建议

验收标准：

- 状态结果具备解释字段。
- 支持查看状态变化历史。

### 5.4 策略/Skill 中心

功能：

- Skill 列表
- Skill 分类
- 版本管理
- 前置条件与特征依赖
- 回测报告
- 仿真表现
- 上线审批状态
- 启停开关

验收标准：

- 任意 Skill 都可查看版本与生效范围。
- Skill 上线必须经过审批和风控校验。

### 5.5 Agent 调度中心

功能：

- 研究 Agent 运行任务
- 信号 Agent 状态
- 执行 Agent 状态
- 风控 Agent 决策日志
- 调度规则与优先级

验收标准：

- 能清晰看到每笔订单由哪个 Agent 和哪个 Skill 触发。

### 5.6 回测与仿真中心

功能：

- 提交回测任务
- 查看收益曲线
- 交易明细
- 风险指标
- 参数对比
- 事件回放
- 仿真盘表现

验收标准：

- 回测结果可复现。
- 支持一键转入仿真模式。

### 5.7 实盘执行中心

功能：

- 账户总览
- 当前持仓
- 活动订单
- 实时 PnL
- 订单执行质量
- 滑点分析
- 人工接管/紧急平仓

验收标准：

- 订单状态与币安账户状态一致。
- 紧急降风险操作优先成功。

### 5.8 风控中心

功能：

- 风控规则配置
- 风险预算
- 单币暴露限制
- 策略限制
- 多空相关暴露图
- 熔断记录
- 告警记录

验收标准：

- 能追踪每次风控拦截的原因和证据。

### 5.9 审计与复盘中心

功能：

- 策略级复盘
- 币种级复盘
- 事件级复盘
- 订单 trace
- 每日/每周/月度报告

验收标准：

- 任意订单可追踪到信号与风险上下文。

## 6. 核心流程

### 6.1 新币种研究接入流程

1. 系统同步新合约元数据。
2. 自动建立基础画像。
3. 拉取历史数据并计算初始特征。
4. 加入扫描池与回测池。
5. 进入可观测列表。

### 6.2 新 Skill 上线流程

1. 研究员提交 Skill 说明。
2. 策略工程师实现并绑定特征。
3. 回测通过。
4. 仿真通过。
5. 风控官审批。
6. 小流量灰度。
7. 正式上线。

### 6.3 自动交易流程

1. 实时行情和事件进入系统。
2. 市场状态引擎更新市场 regime。
3. Signal Engine 评估是否满足某个 Skill 的前置条件。
4. Agent 生成交易意图。
5. Risk Engine 审核。
6. Execution Router 下单。
7. User Stream 回报订单和仓位变化。
8. Portfolio Engine 更新风险暴露和 PnL。
9. 复盘系统归档全过程。

## 7. 非功能需求

- 高可用：关键服务自动重连与异常恢复。
- 高一致：订单状态、仓位、账户需可对账。
- 高透明：全部交易行为可解释。
- 高安全：密钥、权限、审计可控。
- 高扩展：后续可加入更多交易所和事件源。

## 9. 首版 MVP 范围

### 必做

- 币安永续合约元数据同步
- K 线 + Funding + OI + Mark Price 数据采集
- 市场总览页面
- 币种画像页
- Skill Registry V1
- 回测 V1
- 仿真交易 V1
- 风控中心 V1
- 实盘小仓位执行 V1

### 延后

- 多交易所
- 多账户资金调度
- 强化学习自动策略生成
- 高频做市
- 完整移动端

## 10. 验收口径

首版平台完成的标准不是“界面做完”，而是：

- 能稳定采数
- 能稳定研究
- 能稳定回测
- 能稳定仿真
- 能稳定小仓位自动实盘
- 能稳定风控和审计
