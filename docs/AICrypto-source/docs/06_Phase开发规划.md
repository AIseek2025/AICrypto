# AICrypto Phase 开发规划

## 总体原则

- 先打通闭环，再做复杂度。
- 先测试网和仿真，再小仓位实盘。
- 先能力平台，再大规模策略扩张。
- 先风控与审计，再追求策略数量。

## Phase 0：项目奠基

目标：完成工程骨架、开发规范、环境划分、基础设施选型。

交付件：

- Monorepo 结构
- Rust/Python/TypeScript 三栈工程初始化
- 环境变量与密钥规范
- PostgreSQL、Redis、消息总线本地开发环境
- CI 基础流程
- 日志与监控基线

验收标准：

- 本地一键启动开发环境
- 可以跑通最小健康检查链路

## Phase 1：币安接入与数据底座

目标：打通币安永续合约元数据、历史数据、实时流、用户流。

交付件：

- `exchangeInfo` 同步器
- K 线历史补数器
- Mark Price / Funding / OI 采集器
- WebSocket 市场流消费者
- 用户流消费者
- 数据落库和标准化事件协议

验收标准：

- 至少 50 个交易标的稳定采数
- 断线自动恢复
- 数据延迟和丢包可监控

## Phase 2：回测与研究工厂

目标：建立策略研究、特征计算和回测框架。

交付件：

- Feature Warehouse V1
- Strategy Spec 规范
- 回测引擎 V1
- 研究工作流
- 样本标注工具
- 主升浪/主跌浪研究报表

验收标准：

- 可以独立完成一个策略从研究到回测的流程
- 回测结果可复现

## Phase 3：Skill Registry 与 Signal Engine

目标：把研究结论工程化为可运行 Skill。

交付件：

- Skill Registry V1
- 市场状态引擎 V1
- 实时信号评估引擎 V1
- Skill 配置与版本管理
- 候选信号列表页面

验收标准：

- 至少 10 个 Skill 可在历史与实时数据上运行
- 每个 Skill 都有可解释触发原因

## Phase 4：Paper Trading 与风险系统

目标：建立仿真执行和统一风控闸门。

交付件：

- 仿真账户引擎
- 订单状态机
- 风控引擎 V1
- 仿真执行中心页面
- 风险告警中心

验收标准：

- 同一 Skill 可以在回测和仿真环境一致运行
- 能模拟滑点、手续费、挂单未成交和部分成交

## Phase 5：测试网自动化交易

目标：跑通币安测试网的自动实盘闭环。

交付件：

- 测试网交易网关
- 订单对账服务
- 执行质量监控
- 紧急熔断与 reduce-only 通道

验收标准：

- 自动开仓、平仓、止损、撤单全闭环跑通
- 对账正确
- 异常情况有补偿机制

## Phase 6：小仓位生产灰度

目标：在严格风险预算下小仓位运行真实交易。

交付件：

- 生产密钥治理
- 实盘监控中心
- 灰度发布开关
- 策略白名单/黑名单
- 日报、周报和复盘报告

验收标准：

- 不出现失控仓位
- 关键指标稳定
- 出现异常能在秒级触发熔断

## Phase 7：多 Agent 与相关性知识图谱

目标：从单一规则系统升级到多 Agent 协同与相关性交易体系。

交付件：

- 事件接入层
- 社交与新闻解析器
- 相关性图谱 V1
- 多 Agent 调度器 V1
- 相关性策略 Skill 组

验收标准：

- 能对事件驱动场景给出候选交易动作
- 能解释“为什么这个币受该事件影响”

## Phase 8：组合优化与规模化

目标：提高资金利用率、分散风险、提升策略稳定性。

交付件：

- 组合优化器
- 动态资金分配器
- Skill 表现自动降权/升级
- 多账户路由

验收标准：

- 能跨策略做风险预算分配
- 能自动处理策略失效和退化

## 建议节奏

- Phase 0-1：2-3 周
- Phase 2-3：3-5 周
- Phase 4-5：3-4 周
- Phase 6：2-4 周灰度观察
- Phase 7-8：持续迭代

## 核心里程碑

1. 数据底座可用
2. 回测闭环可用
3. Skill 机制可用
4. 仿真闭环可用
5. 测试网闭环可用
6. 实盘灰度可用
7. 相关性策略可用
8. 组合优化可用
