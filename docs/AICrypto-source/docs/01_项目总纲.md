# AICrypto 项目总纲

## 1. 项目定位

AICrypto 是一个面向币安永续合约市场的 AI 自动化投研及交易平台。平台核心目标不是做单一量化脚本，而是建立一套可持续进化的自动化研究、信号发现、策略封装、风险控制、仿真验证与实盘执行体系，重点捕捉主升浪、主跌浪和高波动结构性机会。

## 2. 项目使命

通过自动化采集币安永续合约全市场数据，结合宏观、链上、社交、事件、板块轮动和价格行为等多维信号，识别大级别趋势与高赔率波动机会，并在严格风控下完成自动化交易执行，实现长期可复利的收益增长。

## 3. 核心业务目标

### 3.1 短期目标

- 打通币安 USDⓈ-M 永续合约全自动行情、账户、下单、风控与回报链路。
- 建立永续合约标的主数据库和多频数据湖。
- 完成主升浪/主跌浪研究框架与首批 10-20 个策略 Skill 的工程化封装。
- 建立测试网、历史回测、事件回放、仿真交易、小仓位灰度实盘闭环。

### 3.2 中期目标

- 建立跨币种、跨板块、跨事件源的相关性知识图谱。
- 引入多 Agent 协同研究与交易调度。
- 实现策略自动评估、自动上下线、自动降权与自动切换。
- 构建机构级风险管理与审计系统。

### 3.3 长期目标

- 形成以 Rust 为核心底层的高并发自动化投研交易平台。
- 实现多策略组合、跨市场扩展、跨交易所扩展和多账户资金调度。
- 沉淀 AI 原生策略知识库与可复用 Skill 市场。

## 4. 非目标与边界

- 首版不追求支持所有交易所，先聚焦币安 USDⓈ-M 永续合约。
- 首版不追求超高频做市，重点是分钟级到日级的波段、趋势和事件驱动交易。
- 首版不直接依赖高杠杆盈利，默认以低杠杆甚至无杠杆思路验证策略有效性。
- 首版不把“AI 自由发挥”直接暴露到实盘执行，必须经过规则化和风控网关。

## 5. 产品核心理念

### 5.1 研究驱动执行

平台先做研究工厂，再做交易机器。所有实盘策略都必须来源于可解释、可验证、可回测的研究结论。

### 5.2 Skill 化沉淀

所有可复用的涨跌逻辑、信号组合、仓位计划、止盈止损方法、相关性规则，都要封装成标准化 Skill，供交易 Agent 在满足条件时调用。

### 5.3 事件化与状态机化

市场不是一串指标，而是一系列事件、结构和状态迁移。平台需要把行情、事件、舆情、风控、订单回报都建模为事件流和状态机。

### 5.4 风控优先

研究系统可以激进，实盘系统必须保守。任何策略收益都建立在统一限损、限仓、限频、限相关暴露的前提之上。

## 6. 总体能力地图

平台由 9 大能力域组成：

1. 交易所接入层
   - 币安 REST、WebSocket、User Stream、WebSocket Trading API、测试网。
2. 市场数据层
   - 合约基础信息、K 线、成交、深度、标记价格、资金费率、未平仓量、强平统计、龙虎持仓等。
3. 外部事件层
   - X/Twitter、新闻、宏观事件、名人动态、项目公告、链上异动、板块叙事。
4. 特征与研究层
   - 趋势、波动、相关性、资金行为、情绪、事件冲击、板块轮动、市场状态识别。
5. 策略 Skill 层
   - 入场、加仓、减仓、止盈、止损、撤退、禁开仓、模式切换等标准 Skill。
6. Agent 调度层
   - 研究 Agent、信号 Agent、执行 Agent、风控 Agent、复盘 Agent。
7. 交易执行层
   - 下单路由、订单状态机、滑点控制、失败重试、撤单/平仓/紧急降风险。
8. 风险与治理层
   - 仓位限制、相关暴露控制、黑名单、熔断、权限、审计、告警。
9. 产品交互层
   - 研究工作台、策略中心、实时监控、交易面板、复盘报告、管理员后台。

## 7. 关键成功指标

### 7.1 技术指标

- 行情链路可用性 >= 99.9%。
- 关键 WebSocket 流断连自动恢复时间 <= 5 秒。
- 风控闸门响应延迟 <= 50ms。
- 下单链路可观测率 100%。
- 关键策略回测与仿真结果可复现实验率 >= 95%。

### 7.2 研究指标

- 永续合约可交易标的覆盖率 100%。
- 每个标的至少具备基础特征、资金特征、波动特征和相关性特征。
- 每月新增/迭代 Skill 数量 >= 10。
- 策略从研究结论到仿真上线的平均周期 <= 7 天。

### 7.3 交易指标

- 单策略最大回撤受控。
- 组合级风险暴露在预算内。
- 测试网与仿真环境的订单成功率、状态一致性达到生产要求。
- 实盘灰度期不出现未受控仓位和无限加仓事故。

## 8. 推荐开发语言与技术选型

### 8.1 总原则

- 核心热路径与底层服务：Rust。
- 数据研究、模型实验、特征探索：Python。
- Web 控制台与后台：TypeScript。
- 元数据配置与 Skill 编排：YAML/JSON + PostgreSQL。

### 8.2 技术栈建议

- 核心服务：Rust + Tokio + Axum/Tonic + serde。
- 流处理与队列：Redpanda 或 NATS JetStream；首版可先用 NATS JetStream。
- 数据库：PostgreSQL + TimescaleDB 扩展。
- 缓存：Redis。
- 对象存储：S3 兼容存储，用于原始事件、研究结果、报告。
- 研究计算：Python + Polars + Pandas + PyArrow + Jupyter。
- 回测引擎：Rust 主引擎，Python 研究适配层。
- Web：Next.js + React + TypeScript + Tailwind。
- 身份与权限：Keycloak 或自建 RBAC；首版可先自建简化版。
- 可观测性：Prometheus + Grafana + Loki + Tempo/OpenTelemetry。
- 工作流调度：Temporal 或 Dagster；首版建议 Dagster 做研究任务编排。

## 9. 核心设计约束

- 实盘执行必须与研究环境隔离。
- 风控服务必须独立部署，且有最终否决权。
- 任何 Agent 不能直接持有可无限制使用的交易密钥。
- 任何下单行为必须具备可审计的触发理由、策略 ID、Skill ID、版本号和风险审批上下文。
- 同一逻辑必须同时支持历史回放、仿真交易和实盘执行三种运行模式。
