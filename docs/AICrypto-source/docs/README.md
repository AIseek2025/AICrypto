## 首版推荐实施顺序

1. 先完成 `02_币安永续合约API接入设计.md` 中的测试网连通与行情/下单闭环。
2. 再完成 `03_系统架构设计.md` 中的数据、回测、执行、风控 4 条主链路。
3. 然后按 `05_策略与Skill体系设计.md` 建立第一批可回测、可仿真、可实盘灰度的策略 Skill。
4. 按 `08_开源项目引入设计规划书.md` 持续吸收优秀开源能力，优先嫁接研究层和知识层。
5. 最后按 `06_Phase开发规划.md`、`07_开发工作任务书.md` 和 `09_平台长期建设蓝图.md` 推进整体交付。

## 核心原则摘要

- 核心热路径 Rust-first：实时行情、特征计算、风控、执行、事件总线优先使用 Rust。
- 研究与策略生产双轨：研究探索可用 Python，生产执行必须进入 Rust 核心链路。
- 先测试网、再仿真、后小仓位实盘：禁止未经回测和仿真验证直接全自动实盘。
- 风险控制高于收益追求：任何 Agent 和 Skill 都不能绕过统一风控闸门。
- 一切策略必须可追溯：信号来源、推理过程、参数、订单、PnL 均需完整审计。
- 开源能力持续吸收：优秀开源项目优先通过镜像归档、旁路验证、协议适配和模块嫁接方式融入平台。
