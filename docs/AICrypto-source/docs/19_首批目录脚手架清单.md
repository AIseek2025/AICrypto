# AICrypto 首批目录脚手架清单

## 1. 文档目标

本文件用于把 `18_工程初始化方案.md` 进一步细化成“第一批真正需要创建的目录、占位文件和脚手架说明”。

这份清单的用途是：

- 后续一旦开始编码，可以直接按清单创建骨架
- 保证目录不是随意长出来，而是按架构层次有序落地
- 保证研究层、生产层、协议层、开源归档层从第一天起就是隔离的

## 2. 脚手架总原则

### 2.1 先目录，后实现

首批脚手架主要创建：

- 目录
- README
- 协议说明
- 模块边界说明
- 占位 schema / example 文件

不要求一开始就有完整可运行逻辑。

### 2.2 每个目录必须有说明

所有首批目录都建议至少包含一个 `README.md`，说明：

- 该目录用途
- 对应哪一层
- 允许放什么
- 禁止放什么

### 2.3 协议目录优先于业务目录

因为首版最重要的是保证对象模型稳定。

## 5. 首批必须有示例文件的目录

## 6. 首批必须有 README 的目录

以下目录首批应强制要求 `README.md`：

- 所有顶层目录
- `protocols` 下每个协议目录
- `adapters/exchanges/binance`
- `intelligence/knowledge`
- `intelligence/agents`
- `decision/risk_gateway`
- `execution/order_state`
- `research/experiments`
- `skills` 下每个家族目录

## 7. 建议的创建顺序

按以下顺序创建最稳妥：

1. `protocols`
2. `adapters/exchanges/binance`
3. `intelligence/knowledge`
4. `intelligence/agents`
5. `decision/risk_gateway`
6. `execution/order_state`

## 8. 最终结论

这份脚手架清单的意义是：

- 让下一步工程初始化不再依赖临场发挥
- 让每个模块从第一天开始就有清晰归属
- 让开源能力、研究能力、正式平台能力三者从结构上分离

只要按这份清单落目录，AICrypto 后续正式进入编码时，整体秩序就会非常清楚。
