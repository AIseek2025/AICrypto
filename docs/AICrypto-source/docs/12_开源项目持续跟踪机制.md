# AICrypto 开源项目持续跟踪机制

## 1. 目标

AICrypto 既然要持续吸收全球开源投研、交易、Agent、RAG、图谱与风控生态，就不能靠一次性调研，而必须建立制度化跟踪机制。

本机制用于解决：

- 如何持续发现新项目
- 如何判断哪些项目值得引入
- 如何控制引入风险
- 如何淘汰已经落后的项目

## 2. 跟踪范围

固定跟踪六大池：

1. 交易框架池
2. Binance Futures 专项池
3. 市场数据 / 流处理池
4. Agent / Workflow 池
5. RAG / Graph / Knowledge 池
6. 风控 / 组合 / 回测池

## 3. 信息来源

每次跟踪应优先覆盖以下来源：

- GitHub 搜索
- GitHub Topics / Trending / Updated
- 仓库 README / CHANGELOG / Releases
- PyPI / npm / crates / pkg.go.dev
- 官方 SDK 文档
- 关键项目的 Issues / Discussions / Community 动向

## 4. 跟踪节奏

## 4.1 每周

目标：

- 捕捉新仓库
- 捕捉重点仓库的重大更新

动作：

- 搜索新增仓库
- 检查第一梯队项目是否有重大版本变化
- 补充观察池

输出：

- `weekly radar note`

## 4.2 每月

目标：

- 对重点项目重新评分
- 判断是否需要进入本地归档

动作：

- 复核 A/P/B/C 类项目
- 检查活跃度、版本变动、License、适配价值
- 必要时克隆新仓库到本地

输出：

- `monthly shortlist`

## 4.3 每季度

目标：

- 做引入、升级、淘汰决策

动作：

- 审核每个项目是否：
  - 升级为重点引入
  - 维持观察
  - 降级或淘汰
- 检查已归档项目是否继续值得维护

输出：

- `quarterly open-source review`

## 5. 仓库评分标准

统一从 8 个维度评分：

1. AICrypto 目标贴合度
2. 可插拔性
3. 生产可控性
4. 社区活跃度
5. License 风险
6. 架构借鉴价值
7. 嵌入成本
8. 长期上限

### 分级建议

- `A 级`：必须重点跟踪，适合引入或强参考
- `B 级`：适合专项接入或阶段性借鉴
- `C 级`：观察即可
- `D 级`：淘汰

## 6. 标准决策流程

每个新候选项目统一走以下流程：

1. 发现
2. 初筛
3. 实时联网核验
4. License 审查
5. 本地镜像归档
6. 旁路验证
7. 模块化映射
8. 决定是否接入主平台

## 7. 本地归档策略

### 7.1 何时克隆到本地

满足以下任一条件即可本地归档：

- 进入 A 级
- 对 Binance Futures 有直接工程价值
- 对 Agent / RAG / 图谱有长期架构价值
- 对执行/风控/状态机有明显借鉴价值

### 7.2 本地归档目录

- `external/open_source/<repo>`

### 7.3 归档后要做什么

- 记录用途
- 记录引入边界
- 记录升级频率
- 记录是否已产生嫁接计划

## 8. 引入治理机制

## 8.1 引入前

必须回答：

- 这个仓库解决了什么问题
- 平台已有能力是否已经覆盖
- 它进入后是替代、增强还是旁路
- 它的退出成本有多高

## 8.2 引入中

必须保证：

- 先适配协议
- 先沙箱验证
- 不直接污染核心领域模型
- 不直接获得正式交易权

## 8.3 引入后

必须保留：

- 版本号
- 来源
- 适配层
- 使用范围
- 审计记录

## 9. 淘汰机制

以下情况应考虑降级或淘汰：

- 连续多个周期无维护
- 核心依赖已经过时
- License 风险上升
- 与 AICrypto 路线不再匹配
- 有更优替代品出现
- 接入成本明显高于收益

## 10. 建议的组织执行方式

### 角色分工

- `Open Source Scout`
  - 负责搜集和增量发现
- `Architecture Reviewer`
  - 负责判断是否适合平台化嫁接
- `Risk Reviewer`
  - 负责审查 License、安全、生产边界
- `Integration Owner`
  - 负责落地沙箱验证和模块化嫁接

### 例行产物

- 周报：新增项目与变化
- 月报：评分调整与候选短名单
- 季报：正式引入/淘汰决策

## 11. 建议的自动化方向

后续可以逐步建设自动化开源雷达：

- 定时搜索 GitHub Topics 与关键字
- 抓取仓库 README / Release / Changelog
- 自动提取版本、语言、License、更新日期
- 自动生成候选评分草稿
- 自动写入 AICrypto 的开源观察池

## 12. 对 AICrypto 的直接意义

建立这套机制之后，AICrypto 就不再是“做完一次调研就结束”的项目，而会逐步变成：

- 持续吸收全球开源创新
- 持续把外部能力转成内部模块
- 持续用更先进的项目增强研究、知识和执行能力

这套制度本身，就是 AICrypto 长期竞争力的一部分。
